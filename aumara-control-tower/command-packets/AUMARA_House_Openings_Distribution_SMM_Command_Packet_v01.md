# AUMARA House Openings / Distribution / SMM Command Packet v0.1

## Objective

Open the AUMARA houses for professional guest-facing sale with clean calendars, policies, photography, channel logic, Google visibility and SMM rhythm.

This is not a generic listing task. This is the public opening layer for AUMARA as a hospitality product.

## Strategic Frame

AUMARA should not appear online as random rural accommodation.

AUMARA should appear as a boutique Costa Blanca stay field: houses, silence, light, privacy, mountain/sea access, restaurant connection, retreats, family stays, long weekends and curated recovery.

Core promise:

> Private houses in the quiet inland Costa Blanca, connected to EL CID hospitality, food, nature and a slower rhythm of travel.

## Workstreams

### 1. Inventory and House Naming

Create a clear public inventory map.

Required outputs:
- House list with internal code and public name.
- Capacity per house.
- Bedroom / bathroom count.
- Key features.
- Photo status.
- Calendar status.
- Listing status.

Working house categories:
- Small houses / chalets.
- Superior houses / larger AUMARA homes.
- Any special units tied to retreats, families or longer stays.

### 2. Calendar Opening

Required outputs:
- Master calendar per house.
- Booking.com calendar status.
- Airbnb calendar status.
- Direct booking / internal calendar status.
- Minimum stay rules.
- Blocked dates.
- Cleaning buffer rules.
- Owner / staff use blackout rules.

Initial commercial logic:
- Minimum 2 nights as default.
- Allow 1-night gap fills only where operationally safe.
- Protect key weekends and retreat windows.
- Avoid manual parallel calendars.

### 3. Policies

Required outputs:
- Check-in / check-out policy.
- Cancellation policy.
- Deposit / damage policy.
- Children / extra guest policy.
- Pets policy.
- Smoking / parties / noise policy.
- Pool / garden / common-area rules.
- Cleaning and linen rules.
- Breakfast / restaurant / add-on rules.

Tone:
Clear, warm, firm. Not bureaucratic. Not apologetic.

### 4. Photography and Visual Assets

Required outputs:
- Shot list per house.
- Hero photo selection.
- Room-by-room photo checklist.
- Exterior / terrace / pool / landscape / food / lifestyle shots.
- Before-publish quality gate.
- File naming convention.

Photo hierarchy:
1. Hero exterior / terrace / light.
2. Living / sleeping comfort.
3. Kitchen / bathroom proof.
4. Landscape / pool / silence.
5. EL CID / restaurant / experience layer.

### 5. Listing Copy

Required outputs:
- Short public description.
- Long OTA description.
- Bullet features.
- Guest rules.
- Location copy.
- Experience copy.
- Spanish / English / Russian versions where needed.

Voice:
Boutique, grounded, sensory, not fake luxury.

### 6. Distribution Setup

Required outputs:
- Booking.com listing audit.
- Airbnb listing audit.
- Direct inquiry workflow.
- Price bands.
- Seasonal rules.
- Discount rules.
- Channel commission notes.

Initial rule:
Use distribution to create demand, but keep direct booking path alive from day one.

### 7. Google Business / Maps Visibility

Required outputs:
- Decide whether AUMARA needs separate Google Business Profile or sits under EL CID ecosystem.
- Prepare business name, category, address/service-area logic, photos, description, website/contact link.
- Prepare review request flow.
- Prepare first 10 Google posts / updates.

Positioning rule:
Do not create confusion with EL CID. AUMARA can be a sub-brand, but public user must understand where they arrive, who hosts them, and what they book.

### 8. SMM Launch

Required outputs:
- 14-day launch content plan.
- 30-day evergreen content pillars.
- Instagram / Facebook / Google post copy.
- Short video shot list.
- Founder / host voice posts.
- Guest experience posts.

Content pillars:
- Houses and quiet architecture.
- Morning / breakfast / coffee / terrace.
- Costa Blanca nature and location.
- Family stays.
- Retreat / recovery / women reset.
- Restaurant / Wabi-Sabi / fire kitchen.
- Behind the scenes: opening the field.

### 9. Operational Readiness

Required outputs:
- House readiness checklist.
- Cleaning checklist.
- Maintenance checklist.
- Guest arrival checklist.
- Emergency / contact sheet.
- Welcome message templates.
- Review request templates.

## Definition of Done

AUMARA is considered ready for public opening only when:

1. Every open house has calendar, policy, photos, description and price logic.
2. No house is publicly bookable without cleaning/maintenance readiness.
3. All channels reflect the same availability truth.
4. Guest messages and rules are ready before the first booking.
5. The Google / Maps / SMM layer does not create brand confusion.
6. Owner has a simple daily dashboard: bookings, gaps, blocked dates, issues, reviews.
